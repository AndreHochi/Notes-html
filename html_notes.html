What is an HTML document

An HTML document can be very broadly thought of as a tree, an example can be found clicking here.

Starting a HTML document

Every HTML document requires these three things.

  1. Every HTML document requires <!DOCTYPE html> in the start this indicates that the document is an HTML5 document.
  2. Every HTML document requires <html> is the root element of the HTML page. Consider this as the starting element that will wrap around everything that is in the HTML language, and indicate everything inside is HTML.
  3. Every HTML document requires </html> is the root element of the HTML page. Consider this as the ending element that will wrap around everything that is in the HTML language, and indicate everything inside is HTML.

So at least in my eyes the simpliest HTML5 document is below.

Example

<!DOCTYPE html>
<html>
</html>

This document has nothing in it, but it will be interpreted as a html5 document with a html root.

HTML Elements

HTML documents are built off of HTML Elements, infact you have seen one already, the HTML element. Everything between <html> and </html> is an HTML element. HTML elements very broadly can be split in to two categories, nested elements and empty (unnested) elements.

Nested Elements

You can consider nested elements as branches of the tree. Nested elements can contain other elements inside them. For example the <html> element contains every other element. Nested elements must contain one thing but may contain four things. Lets use the element <p> as an example. <p> just designates a paragraph element.

  1. Nested elements must contain a starting tag. ex) <p>
  2. Nested elements generally contain a ending tag. (Sometimes this can be omitted, but this is bad practice because this is not consistent) ex) </p>
  3. Nested elements generally wraps around text. ex) <p>Hello.</p>
  4. Nested elements can contain attributes. ex) <p style="color:red">Hello.</p>

Below are a list of some common nested elements.

  • <!DOCTYPE> This defines the document type. There are a few different types of HTML (HTML5, HTML4, XHTML) and this will define which type you will be using. <!DOCTYPE html> by default uses HTML5.
  • <html> This is the root element for the entire HTML document. A notible attribute to this is you can define the language of your document to make it easier for search engines to find your document for example < html lang="en">.
  • <head> This the head element. This defines alot of metadata about the document. This is where you would define <title>, <style>, and <link>.
  • <title> This is the title element. This defines the title in the tab of your window. This also helps search engines find your website.
  • <style> This is the style element. This is where you can define all the styling for visible elements. Below is an example of some styling.

    <style>
    h1 {color:red;}
    </style>

  • <h1> to <h6> This is the header element. This is where you write out headers. <h1> is by default the largest header and <h6> is by default the smallest header. This element supports global attributes.
  • <p> This is the paragraph element. This is where you would write out paragraphs. This is likely going to be the bulk of the visible written element seen by the user. This element supports global attributes.
  • <b> This is the bold element. This element when wrapped around another text element will turn the text bold.
  • <a> This is the hyperlink element. This element when wrapped around another text element will add a hyperlink to them. This element has an attribute called href which needs to be assigned to the linked website
  • <ul> This is the unordered list element. This element defines an unordered list, text bold.
  • <ol> This is the ordered list element. This element defines an ordered list, text bold.
  • <li> This is the list item element. This element wrapped around by the <ul> or <ol> element will add the item to the list. text bold.

Empty (Unnested) HTML Elements

Empty (Unnested) HTML Elements are elements that do not contain other elements. For example an <img> element will not contain other elements. <img> is an image element. Empty (Unnested) HTML Elements generally contain two things.

  1. Empty (Unnested) elements must contain a starting tag. ex) <img>
  2. Empty (Unnested) elements can contain attributes. ex) <img src="img_girl.jpg">

Below are a list of some common empty (unnested) elements.

  • <link> This is the link element. Despite the name this is not a hyperlink that will take you to a new website this is a link to a css document that you can use to style everything instead of using the style element (this is also the prefered way of styling).
  • <img> This is the image element. There are four notable attributes, src which stands for source where you provide the source to the image, alt which displays the alt text for the image, and width and height which displays the width and height of the image.

Global Attributes

Global attributes are attributes used in all HTML elements. Whether its worthwhile is another question.

  • class. This attribute defines the class of an object. This is normally used for styling to define alot of objects as the same class so the styling can be applied to all of them
  • id. This attribute defines a unique ID for an object. This is normally used with javascript.
  • lang. This attribute defines the lanaguage of an object.
  • style. This attribute defines inline css for an object.
  • title. This attribute defines extra information for an object. This information is normally seen when hovering over the object.
